**1) Step1Basic — title Field (line 383-393):**

기존:
```tsx
<Field label="공고 제목" required hint="10자 이내로 핵심만">
  <input type="text" ... maxLength={40} ... />
</Field>
```

변경:
- `hint` 문구를 `"4자 이상, 10자 이내로 핵심만"` 으로 바꾼다 (최소값 노출).
- `<input>` 바로 뒤(Field children 안, closing `</Field>` 앞)에 Step 3 패턴과 동일한 인라인 경고를 추가한다:
```tsx
{form.title.trim().length > 0 && form.title.trim().length < 4 && (
  <p className="text-[10px] font-bold text-destructive">
    제목은 최소 4자 이상 입력해주세요
  </p>
)}
```

**2) Step1Basic — description Field (line 416-426):**

기존 `hint="업무 내용, 분위기 등 자유롭게"` 를 `"10자 이상, 업무 내용·분위기 등 자유롭게"` 로 바꾼다.

`<textarea>` 바로 뒤에 인라인 경고 추가:
```tsx
{form.description.trim().length > 0 && form.description.trim().length < 10 && (
  <p className="text-[10px] font-bold text-destructive">
    업무 소개는 최소 10자 이상 입력해주세요
  </p>
)}
```

**3) Step1Basic — category Field (line 395-414):**
category는 "입력 중" 상태가 없어(클릭=선택 완료) Step 3 스타일의 "progressive" 경고가 부적합하다. 대신 기존 `required *` 빨간 별표로 충분하다는 RESEARCH 판단을 유지하되, Field `hint` 로 보조 안내만 추가한다:
```tsx
<Field label="카테고리" required hint="업무 유형을 하나 선택해 주세요">
```
(인라인 빨간 경고는 추가하지 않음 — 사용자 눌렀는데 아무 반응 없는 상황 아니고 그리드 자체가 시각적으로 요구함.)

**4) Step2Schedule — 근무일 Field (line 454-462):**

`<input type="date">` 바로 뒤에 인라인 경고 추가 (단, Step 2 진입 후 최초 렌더에서 일괄적으로 경고 폭탄이 뜨지 않도록, "다른 Step 2 필드 중 하나라도 입력 시작됐을 때만" 표시 → 이는 과도하게 복잡. RESEARCH의 권장대로 **빈 상태는 조용히 required *만 표시**, 날짜 필드는 HTML5 date picker 특성상 "비어있음"이 유일한 실패 모드이므로 경고 대신 `hint` 로 명시):

변경: `<Field label="근무일" required>` → `<Field label="근무일" required hint="근무할 날짜를 선택해 주세요">`

**5) Step2Schedule — 시작 시간 / 종료 시간 (line 464-481):**
