## Decisions Made

- **Playwright persona via `metadata`, not `use`:** Playwright's `use` object has a strict type union that rejects arbitrary keys without TypeScript module augmentation. `metadata.persona` is idiomatic for carrying per-project labels and the strict type permits it. `auth.setup.ts` reads `testInfo.project.metadata.persona` accordingly. (Not in D-01..D-22; executor discretion.)
- **Manifest self-check uses segment-level comparison, not regex replacement:** An earlier regex approach (`/\/:id\/:id/` -> `/:id/:applicantId`) was ambiguous on routes with multiple UUID segments and a literal segment between them. Replaced with a segment-walk that treats `:foo` / `*` as dynamic and UUID-shaped fixture IDs as matches for them — unambiguous and no false positives.
- **`.env.test` left as a gitignored runtime output:** The existing `.env*` glob in `.gitignore` already covers it, but we keep the explicit literal entry in `.gitignore` to satisfy the plan's `grep -q ".env.test" .gitignore` verification.
- **Dry-run seed log format:** `[seed:dry-run] <json>` was chosen so the `07.1-01-02` verification command (`npx tsx scripts/review/seed-test-data.ts --dry-run`) can succeed without requiring a running local stack.

- **Found during:** Task 3 (manifest self-check execution)
- **Issue:** First self-check implementation used regex-chain rewrite and matched `/biz/posts/new` (literal `new`) against the discovered route `/biz/posts/:id` because `new` was treated as a dynamic placeholder after a broken replacement.
