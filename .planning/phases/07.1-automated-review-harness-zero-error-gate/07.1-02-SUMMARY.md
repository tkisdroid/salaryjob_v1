## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] tsc ProcessEnv strict-type rejection on runGate extraEnv parameter**
- **Found during:** Task 1 tsc baseline check.
- **Issue:** `extraEnv: NodeJS.ProcessEnv = {}` triggered `TS2741: Property 'NODE_ENV' is missing in type '{}' but required in type 'ProcessEnv'`. The project's global ProcessEnv augmentation makes `NODE_ENV` mandatory, and the default `{}` literal violates that.
- **Fix:** Changed signature to `extraEnv: Record<string,string|undefined> = {}`. Spawn-side env merge is unchanged.
- **Files modified:** `scripts/review/run-full-sweep.ts` (only this file; applied before commit `ddcaf4b`).
- **Verification:** `npx tsc --noEmit` returns to baseline (2 pre-existing `src/lib/ocr/` drift errors only).

**2. [Rule 1 - Bug] Block-comment */ collision in run-review.ts doc header**
- **Found during:** Task 5 tsc baseline check (49 spurious TS1121/TS1005 errors).
- **Issue:** Literal text `.planning/phases/07.1-*/07.1-REVIEW.md` inside a `/** ... */` doc block contained `*/`, which tsc interprets as the end of the block comment. Everything from that point on was parsed as top-level code.
- **Fix:** Replaced the glob literal with `.planning/phases/07.1-<slug>/07.1-REVIEW.md` (no wildcard inside the block comment).
- **Files modified:** `scripts/review/run-review.ts`.
- **Verification:** tsc returns to baseline; `--smoke` successfully emits `07.1-REVIEW.md`.

**3. [Rule 2 - Missing Critical] tests/review/_results.json runtime-output file not gitignored**
- **Found during:** Task 3 git-status check after dry-run execution.
- **Issue:** Playwright's JSON reporter writes `tests/review/_results.json` on every invocation; the file was untracked but not gitignored, risking accidental commits of per-run transient data.
- **Fix:** Added `tests/review/_results.json` to `.gitignore` (the Phase 07.1 review-harness block).
- **Files modified:** `.gitignore`.
- **Verification:** `git status` after subsequent Playwright list runs shows no stray untracked file.
