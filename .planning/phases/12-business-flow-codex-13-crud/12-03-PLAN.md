**BUG-B13: Category-filtered worker list**

In `src/app/biz/workers/page.tsx`:

1. After `const session = await requireBusiness();` (line 11), add a query to get the business's unique job categories:
```typescript
const businessJobs = await prisma.job.findMany({
  where: { authorId: session.id },
  select: { category: true },
  distinct: ["category"],
});
const businessCategories = businessJobs.map((j) => j.category);
```

2. Replace the `prisma.workerProfile.findMany` call inside the `Promise.all` (lines 14-17):

FROM:
```typescript
prisma.workerProfile.findMany({
  include: { user: { select: { id: true, email: true } } },
  orderBy: [{ rating: "desc" }, { totalJobs: "desc" }, { createdAt: "desc" }],
  take: 100,
}),
```

TO:
```typescript
prisma.workerProfile.findMany({
  where: businessCategories.length > 0
    ? { preferredCategories: { hasSome: businessCategories } }
    : {},
  include: { user: { select: { id: true, email: true } } },
  orderBy: [{ rating: "desc" }, { totalJobs: "desc" }, { createdAt: "desc" }],
  take: 100,
}),
```

This filters workers whose `preferredCategories` array overlaps with the business's job categories. If the business has no posted jobs (empty categories), the `where: {}` fallback shows all workers so new businesses are not stuck with an empty list.

The `preferredCategories` field is a `String[]` array column on the `WorkerProfile` model, so Prisma's `hasSome` operator is the correct filter for array overlap.

Note: The `Promise.all` block needs the `businessJobs` query to resolve first, so move the `Promise.all` after the `businessCategories` computation. The updated structure:
```typescript
const businessJobs = await prisma.job.findMany({
  where: { authorId: session.id },
  select: { category: true },
  distinct: ["category"],
});
const businessCategories = businessJobs.map((j) => j.category);

const [profiles, favorites] = await Promise.all([
  prisma.workerProfile.findMany({
    where: businessCategories.length > 0
      ? { preferredCategories: { hasSome: businessCategories } }
      : {},
    include: { user: { select: { id: true, email: true } } },
    orderBy: [{ rating: "desc" }, { totalJobs: "desc" }, { createdAt: "desc" }],
    take: 100,
  }),
  prisma.favoriteWorker.findMany({
    where: { businessUserId: session.id },
    select: { workerId: true },
  }),
]);
```
