**BUG-W12: role-select/actions.ts**

The `selectRole` action does `prisma.user.update({ where: { id: session.id } })` which will throw a `RecordNotFound` error if the user row does not exist in the public.users table. Social/magic-link signup flows bypass `signUpWithPassword` and its `createInitialProfile` helper, so the prisma user row may not exist.

Fix: Change `prisma.user.update` (line 25-28) to `prisma.user.upsert`:

```typescript
// BEFORE:
await prisma.user.update({
  where: { id: session.id },
  data: { role: parsed.data },
})

// AFTER:
await prisma.user.upsert({
  where: { id: session.id },
  create: {
    id: session.id,
    email: null,
    role: parsed.data,
  },
  update: { role: parsed.data },
})
```

Then add profile bootstrapping after the upsert (before the admin metadata update). Insert after the upsert block:

```typescript
// BUG-W12: Ensure profile exists for social/magic-link users who skip signUpWithPassword
if (parsed.data === 'WORKER' || parsed.data === 'BOTH') {
  const existingProfile = await prisma.workerProfile.findUnique({
    where: { userId: session.id },
    select: { id: true },
  })
  if (!existingProfile) {
    await prisma.workerProfile.create({
      data: {
        userId: session.id,
        name: '사용자',
        preferredCategories: [],
      },
    })
  }
}
```

This ensures social login users always have the necessary prisma rows before entering the app.
