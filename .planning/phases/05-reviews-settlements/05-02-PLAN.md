<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Prisma schema ↔ Supabase Postgres enum | Additive enum change via `prisma db push`. Non-transactional on the Postgres side (ALTER TYPE cannot be rolled back in a migration block in older versions). |
| Client-submitted review input ↔ Server Action | Zod validation at the entry point of all review Server Actions (Plan 03). Phase 5 Plan 02 prepares the schemas; Plan 03 invokes them. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | Review rating input (src/lib/validations/review.ts) | mitigate | Zod `z.number().int().min(1).max(5)` rejects rating=0/6/-1/3.5/NaN before ever reaching the DB. |
| T-05-05 | Tampering | Review comment XSS (src/lib/validations/review.ts) | mitigate | Zod `z.string().max(500)` length cap. React auto-escapes `{comment}` in JSX (no dangerouslySetInnerHTML planned). Defense-in-depth. |
| T-05-08 | Tampering | Review tags array (src/lib/validations/review.ts) | mitigate | Zod `z.array(z.string().max(20)).max(8)` validates shape before Prisma parameter binding. Prisma Client parameterizes text[] — SQL injection blocked at ORM layer. |
| T-05-21 | Denial of Service | prisma db push enum change | accept | Additive enum changes in Postgres 15 are fast metadata operations. Phase 4 verified the same workflow when adding `pending`. Rollback plan: `ALTER TYPE ... DROP VALUE` is not supported by Postgres, but the value can remain unused if the plan needs rollback. |
| T-05-22 | Information Disclosure | Error codes leaking internals | mitigate | reviewErrorToKorean uses an exhaustive switch with user-safe messages only — raw exception details never reach the client (mirrors the Phase 4 T-04-21 mitigation). |
</threat_model>
