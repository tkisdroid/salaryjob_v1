# 샐러리잡 디자인 브리프

## 프로젝트 목표

`샐러리잡`은 지역 기반 일자리 플랫폼이지만, 첫인상은 구인구직 사이트보다 `가깝고 믿을 수 있는 생활 서비스`에 더 가까워야 한다.  
디자인 방향은 당근의 친근함, 쉬운 정보 구조, 넉넉한 여백, 큰 메시지 중심 구성을 참고하되, 그대로 복제하지 않고 `샐러리잡만의 초록색 브랜드 톤`으로 재구성한다.

핵심 인상은 아래 3가지다.

- 신뢰감 있는 로컬 일자리 플랫폼
- 복잡하지 않은 프로페셔널 UI
- 신선하고 가벼운 셀러리(샐러리) 같은 초록 무드

## 벤치마킹 해석

당근에서 참고할 개념은 아래와 같다.

- 큰 헤드라인과 짧은 설명으로 바로 이해되는 첫 화면
- 여백이 넓고 정보 위계가 분명한 랜딩 구조
- 카드 중심 UI와 둥근 모서리로 부담 없는 인상
- 서비스가 어렵지 않다는 느낌을 주는 친근한 톤
- 기능보다 `생활 속 문제 해결`을 먼저 보여주는 구성

샐러리잡에서는 이를 아래처럼 바꿔 적용한다.

- `중고거래/동네생활` 느낌 대신 `내 주변 일자리/오늘 가능한 일/빠른 매칭`
- 오렌지의 따뜻함 대신 `산뜻한 그린` 기반의 신뢰감
- 커뮤니티 느낌은 유지하되, 더 정돈된 B2C/B2B 서비스 톤으로 업그레이드
- 캐주얼함 60%, 프로페셔널함 40% 비율 유지

중요:  
당근의 레이아웃을 그대로 복사하거나 아이콘, 일러스트, 카피 구조를 모사하지 않는다.  
`친근한 생활형 플랫폼`이라는 설계 철학만 참고하고, 결과물은 샐러리잡의 독립 브랜드처럼 보여야 한다.

## 브랜드 방향

### 브랜드 한 줄 정의

`내 주변 일자리를 가장 쉽고 산뜻하게 연결하는 로컬 잡 플랫폼`

### 톤 앤 무드

- 밝고 깨끗하다
- 믿을 수 있다
- 과하게 기술적이지 않다
- 모바일에서도 편하다
- 구인자와 구직자 모두에게 부담이 없다

### 피해야 할 인상

- 어두운 채용 플랫폼
- 보라색/남색 위주의 스타트업 템플릿
- 복잡한 대시보드형 첫 화면
- 과한 그라데이션, 과한 3D, 과한 애니메이션
- 금융앱처럼 딱딱한 분위기

## 컬러 시스템

메인 색은 `셀러리처럼 신선한 초록색`으로 잡고, 나머지 색도 초록 계열과 자연스럽게 어울리는 범위만 사용한다.

### 메인 팔레트

- Primary: `#41B66E`
- Primary Hover: `#2F9E59`
- Primary Soft: `#EAF8EE`
- Primary Deep: `#1E6F3D`
- Mint Background: `#F5FBF6`
- Lime Accent: `#B7D77A`
- Warm Ivory: `#FFFDF8`
- Text Strong: `#1E2A22`
- Text Body: `#4D5B52`
- Border Soft: `#DCEADF`

### 사용 원칙

- 화면 전체는 화이트 또는 아이보리 베이스
- 진한 초록은 CTA, 활성 상태, 핵심 수치에만 사용
- 연한 민트는 섹션 배경, 카드 하이라이트, 배지 배경에 사용
- 라임 계열은 서브 포인트로만 제한적으로 사용
- 빨강/보라/강한 파랑은 경고 상황 외에는 사용하지 않음

## 타이포그래피

### 추천 폰트

- 한글: `Pretendard`, `SUIT`
- 영문 숫자 보조: `Inter` 또는 시스템 sans

### 규칙

- Hero 제목: 크고 짧고 강하게
- 본문 카피: 짧고 읽기 쉽게
- 카드 제목: 18~22px 수준의 명확한 위계
- 설명문: 한 줄 길이를 짧게 유지
- 숫자/금액/시간은 시각적으로 더 강하게

### 타이포 인상

- 귀엽게 보이는 폰트 금지
- 지나치게 기술적인 monospace 느낌 금지
- 제목은 단단하게, 본문은 부드럽게

## 레이아웃 원칙

### 공통 구조

- 넓은 여백
- 최대 폭이 정돈된 중앙 정렬
- 한 섹션마다 목적이 하나만 드러나게 구성
- 카드와 배지를 활용해 정보 블록을 짧게 끊기

### 첫 화면 구성

1. 상단 네비게이션
2. 강한 메시지의 히어로 섹션
3. 근처 일자리/핵심 장점 카드
4. 서비스 흐름 설명
5. 구직자용 가치
6. 사업자용 가치
7. 실제 후기 또는 신뢰 수치
8. 최종 CTA

### 화면 밀도

- 정보는 많아 보이지 않게
- 스크롤은 허용하되, 각 섹션마다 `왜 봐야 하는지` 즉시 이해되어야 함
- 한 화면에 너무 많은 카드 금지

## 메인 페이지 섹션 가이드

## 1. 헤더

- 좌측: 샐러리잡 로고
- 우측: 일자리 찾기 / 사업자 등록 / 로그인 / 시작하기
- 모바일에서는 메뉴 단순화
- 배경은 흰색 또는 반투명 화이트
- 스크롤 시 아주 약한 블러 또는 얇은 보더만 적용

## 2. 히어로

### 핵심 메시지 예시

- `내 주변 일자리, 더 가볍고 빠르게`
- `오늘 가능한 일부터 안정적인 근무까지`
- `샐러리처럼 산뜻한 로컬 잡 플랫폼`

### 구성

- 좌측: 짧은 헤드라인 + 서브카피 + CTA 2개
- 우측: 앱/웹 UI 프리뷰 카드 또는 일자리 카드 묶음
- 배경은 아주 연한 민트/아이보리 톤
- 과한 일러스트보다 실제 제품 UI 느낌이 더 좋음

### CTA

- Primary: `내 주변 일자리 보기`
- Secondary: `사업자로 시작하기`

## 3. 핵심 가치 카드

3~4개 카드로 간결하게 보여준다.

- 빠른 지원
- 근처 일자리 매칭
- 명확한 급여/시간 정보
- 신뢰 가능한 사업장 정보

카드는 둥글고 밝아야 하며, 그림자보다 보더와 배경 대비로 정리한다.

## 4. 서비스 흐름

구직자 기준 3단계 혹은 4단계

1. 근처 일자리 탐색
2. 조건 확인 후 빠른 지원
3. 근무 진행
4. 정산/리뷰 확인

당근처럼 이해가 쉬운 방식이 중요하므로, 복잡한 다이어그램보다 짧은 단계 카드가 적합하다.

## 5. 구직자 중심 섹션

- 오늘 가능한 일
- 집 가까운 일
- 급여가 명확한 일
- 평판이 좋은 사업장

이 섹션은 `실용성`이 핵심이다.  
감성보다 정보가 먼저 보여야 한다.

## 6. 사업자 중심 섹션

- 빠른 인력 확보
- 지원자 관리
- 근무/정산 관리
- 반복 채용 효율화

구직자 섹션보다 조금 더 정돈되고 신뢰감 있게 보이도록 설계한다.

## 7. 후기 또는 신뢰 지표

- 사용자 후기
- 평균 매칭 시간
- 등록된 사업장 수
- 누적 지원 수

숫자는 크고 명확하게 보여주되, 배경은 과하지 않게 유지한다.

## 8. 마지막 CTA

- `지금 샐러리잡 시작하기`
- `가까운 일자리 찾기`
- `사업장 등록하기`

마지막 CTA 섹션은 메인 그린 컬러를 적극적으로 사용하되, 텍스트 대비를 명확히 유지한다.

## 컴포넌트 스타일 가이드

## 버튼

- Primary: 진한 초록 배경, 흰 텍스트
- Secondary: 연한 민트 배경, 진한 초록 텍스트
- Ghost: 흰 배경, 얇은 보더
- radius는 충분히 둥글게
- hover는 색을 진하게만 바꾸고 효과는 절제

## 카드

- 둥근 모서리 20~28px
- 밝은 배경
- 약한 보더
- 그림자는 아주 약하게
- 카드 내부 여백은 넉넉하게

## 배지

- 급구, 추천, 근처, 인기 같은 태그에 사용
- 작은 사이즈
- 컬러는 primary soft 기반

## 입력 폼

- 구인구직 서비스 특성상 폼이 많으므로 단정하고 명확해야 함
- 높이는 충분히 크게
- placeholder는 연하게
- 포커스 시 초록색 테두리 + 약한 글로우

---

## 이미지 및 그래픽 방향

- 실제 서비스 화면처럼 보이는 UI 목업 위주
- 사람 사진을 쓸 경우 지나치게 광고 느낌 나는 스톡 이미지는 피함
- 일러스트를 쓰더라도 너무 귀엽거나 유치하지 않게
- 아이콘은 라인 기반, 깔끔한 스타일 유지

---

## 모바일 원칙

- 모바일에서 가장 먼저 완성되어야 함
- 히어로는 세로 스택 구조
- CTA는 한눈에 보여야 함
- 카드 텍스트는 2줄 이상 길어지지 않게
- 하단 고정 CTA가 필요하면 과하지 않게 사용
- 정보가 많은 섹션은 슬라이더보다 세로 카드가 더 안정적

---

## 인터랙션 원칙

- 애니메이션은 짧고 부드럽게
- hover보다 scroll reveal, fade-up 정도만 사용
- 인터랙션은 존재감보다 완성도를 높이는 역할
- 과한 bounce, scale, flashy motion 금지

---

## 카피라이팅 원칙

- 짧고 직접적
- 쉬운 단어 사용
- 기능 설명보다 결과 중심
- 지역성, 속도, 신뢰를 강조

### 톤 예시

- `가까운 일자리부터 바로 확인하세요`
- `복잡하지 않게, 필요한 일만 빠르게`
- `내 주변에서 찾는 가장 쉬운 일자리`
- `사업장에 맞는 인력을 더 빠르게 연결합니다`

## Lovable 작업용 지시문

아래 문장을 Lovable 디자인 작업 지시문으로 바로 사용해도 된다.

```md
샐러리잡이라는 로컬 일자리 플랫폼 웹사이트를 디자인한다.
전반적인 구조감은 당근의 친근하고 쉬운 정보 구조를 참고하되, 그대로 복사하지 말고 독립적인 브랜드처럼 재해석한다.
핵심 컨셉은 산뜻한 셀러리 그린 기반의 프로페셔널하고 심플한 서비스 디자인이다.

메인 컬러는 신선한 초록색 계열로 하고, 보조 색상도 초록과 자연스럽게 어울리는 민트, 라임, 아이보리, 딥그린 정도로만 제한한다.
보라, 강한 파랑, 과한 그라데이션, 무거운 다크톤은 사용하지 않는다.

첫 화면은 큰 헤드라인, 짧은 설명, 명확한 CTA 2개, 그리고 제품 UI 카드 프리뷰로 구성한다.
전체적으로 여백이 넉넉하고, 카드 기반이며, 라운드가 부드럽고, 신뢰감 있는 로컬 플랫폼 인상이 나야 한다.

섹션 구성은 다음 순서로 한다:
헤더 → 히어로 → 핵심 가치 카드 → 서비스 이용 흐름 → 구직자 가치 → 사업자 가치 → 후기/신뢰 지표 → 마지막 CTA

모바일에서도 깔끔해야 하며, UI는 복잡한 채용 플랫폼보다 더 친근하고 쉽게 느껴져야 한다.
디자인은 세련되고 전문적이어야 하지만 차갑지 않아야 한다.
```

## 최종 디자인 판단 기준

아래 질문에 모두 `예`가 나오면 방향이 맞다.

- 첫 화면 3초 안에 어떤 서비스인지 이해되는가?
- 초록색이 신선하고 세련되게 보이는가?
- 친근하지만 가볍지 않고, 서비스 신뢰감이 있는가?
- 당근의 느낌은 참고했지만 그대로 닮아 보이지는 않는가?
- 모바일에서 더 좋아 보이는가?
- 구직자와 사업자 모두가 쉽게 이해할 수 있는가?
