## Design Context

### Users

**Primary surface — Worker (모바일, 이동 중)**
- 20대 중반 ~ 40대 중반, 서울/수도권 도심 출퇴근·주말 시간에 **돈 1~10만 원** 단위로 빈 시간을 채우려는 사람.
- 이력서·면접 과정을 견디기 싫어서 기존 알바천국/알바몬을 이미 졸업했거나 애초에 쓰지 않는 사람.
- 사용 컨텍스트: 지하철/버스/카페에서 한 손으로 폰 조작 · 30초 안에 "오늘 갈 수 있는 일" 탐색 · 그대로 원탭 지원 · 현장 도착 후 QR 체크인 · 근무 끝나면 앱으로 즉시 정산 확인.
- **은퇴자·투잡 직장인** 은 부수적 관심사 — 카피가 그쪽으로 기울면 primary 경험이 흐려짐 (이미 이 세션에서 확인된 사실).

**Secondary surface — Business (데스크톱, 업무 중)**
- 카페·편의점·물류센터·행사장 운영자. "지금 한 명 당장 필요" 또는 "이번 주말 3명" 이 가장 잦은 use case.
- 사용 컨텍스트: 매장 카운터 옆 노트북 · 사무실 모니터 · 태블릿. 데이터 밀도 높은 관리 UI 선호, 모바일은 긴급 대응용 보조.

**Job to be done (one-line)**:
> Worker 는 "이 근처에 지금 내가 할 수 있는 일이 있나?" 를 30초 안에 답받고 탭 한 번으로 확정하고 싶다.
> Business 는 "지금 한 명 필요한데, 면접 없이 믿을 만한 사람을 몇 분 안에 채우고 싶다."

### Brand Personality

**Three words**: **산뜻한 · 꾸밈없는 · 즉답하는**

- **산뜻한 (fresh, clean)** — 제품 이름부터 시각 언어까지 이 축이 brand spine. 무겁거나 진중한 톤은 이 제품에 맞지 않음. 칙칙한 채용 플랫폼과의 가장 뚜렷한 대비점.
- **꾸밈없는 (unadorned, honest)** — 이 세션에서 확정: 사용자는 "샐러리 한 움큼 / 셀러리처럼 산뜻한 또 하나의 샐러리" 같은 pun-heavy 카피를 거부하고 "내 주변 일자리, 더 가볍고 빠르게" 같은 기능적·직설적 카피를 선택했음. 말장난 > 기능. 수식어 > 서술어. → 수식어 뺀다.
- **즉답하는 (responsive, immediate)** — 제품의 전 경험이 "탭 한 번 → 즉시 확정 → 즉시 정산" 이라는 하나의 약속을 반복한다. UI 도 대기·로딩·중간 설명을 최소화해야 함. 버튼은 누르는 순간 상태가 바뀌고, 서버 왕복은 optimistic UI 로 가린다.

**Emotional priority (순서가 중요함)**:
1. **가벼움 (ease)** — 제품 존재 이유 자체. 모든 디자인 결정은 "이게 가볍게 느껴지나?" 로 먼저 검증.
2. **안심 (trust)** — 초단기 근무·즉시 정산 모델은 "돈 떼이지 않는다" 는 신뢰가 전제. 시각 언어에서 돈/정산 관련 화면은 특히 단단해야 함.
3. **기회 (opportunity)** — 급구·긴급 모집 같은 도파민 훅은 가지고 있지만 주된 정서는 아님. urgency 는 배지·색 한 점으로만 표현.

### Aesthetic Direction

**Reference (이런 느낌이었으면)**
- **Toss (토스)** — 한국 사용자에게 "돈/금융을 이렇게 가볍게 다룰 수도 있구나" 를 각인시킨 제품. 민트 계열 팔레트 · 아이콘 기하 · 큰 숫자 타이포 · 걷어낸 설명 · 원탭 transaction 프레임. 샐러리잡은 "금액이 오가는 모바일 앱" 이라는 속성상 Toss 와 가장 가까운 감각을 지향한다.
- **당근마켓** — 하이퍼로컬 + 친절하되 유치하지 않음 + 한국어 타이포 리듬. "내 주변" 을 전제로 한 UX 플로우 (거리 · 동네 단위) 가 동일.
- **Linear (linear.app)** — 해외 레퍼런스. 정교한 micro-interaction · 조용한 모션 · 타이포 기반 위계 · 과장되지 않은 브랜드. Biz 관리 대시보드 톤은 이쪽에 가까움.

**Anti-reference (이렇게는 절대 아님)**
- **알바천국 / 알바몬** — 샐러리잡이 **딱 거스르려는 제품**. 이력서 중심, 채용 담당자 시선, 정보 밀도 과잉, 배너 광고, 브랜드 색 통일성 없음. 이 제품들과 *한 화면 안에서 봤을 때 즉시 다른 세계관임을 인지시켜야* 한다.
- **사람인 / 잡코리아** — 경력·커리어 채용 UX. "면접" 이 중심에 있는 제품군. 샐러리잡은 "면접 없음" 이 상품이기 때문에 이 톤을 하나라도 닮으면 제품 정체성이 흔들림.

**Theme**
- **Light only, mint-forward.** 다크모드 추가 계획 없음.
  - *Why:* 사용자는 주로 **낮 시간 이동 중 (통근·점심·주말 오전)** 에 "지금 할 일 있나?" 를 검색한다. 카페·버스 안 고휘도 환경이 지배적. 다크모드는 밤중 음악 앱·트레이딩 대시보드의 맥락이지 gig 마켓의 맥락이 아님.
  - 추후 QA 표면과 토큰 세트가 배로 늘어나는 비용도 명시적으로 거부.
  - light 기반이되 순백(`#ffffff`) 이 아닌 ivory/mint 계열 (`--mint-bg: #f6fcf8`) 위에서 운영 — 자연에 순백은 없다는 impeccable 원칙과 일치.

### Palette (확정, 세션에서 튜닝 완료)

```css
--brand:        #5bc98a;   /* Primary — soft fresh green */
--brand-dark:   #42b474;   /* Hover, pressed */
--brand-light:  #ecfaf0;   /* Badge · card tint · selected bg */
--brand-deep:   #2b8a55;   /* Heavy accent, outline text on light surface */
--lime-accent:  #c7e295;   /* Sub-accent, secondary chips */
--teal:         #44b5a6;   /* Paired accent — harmonises with mint brand */
--teal-light:   #dcf2ed;
--mint-bg:      #f6fcf8;   /* Section fill, near-white mint */
--ivory:        #fffdf8;   /* Warm ivory for the most neutral surface */
```

**60-30-10 분배**
- **60%** — ivory/mint-bg 중립 표면 + Pretendard 본문
- **30%** — muted-foreground 본문 보조 텍스트, border, subtle chrome
- **10%** — `--brand` 초록 하나만. Brand green 은 **강조 1회/화면** 룰을 지켜야 "산뜻함" 이 유지됨. 화면 전체가 초록이면 "건강식품 브랜드" 로 미끄러짐.

**Tint neutrals toward brand**: impeccable color_principles 에 맞춰 neutral 토큰은 brand hue 쪽으로 아주 약하게 틴팅되어 있어야 함 (`oklch(0.97 0.005 150)` 류). 현재 globals.css 값은 hex 라서 추후 OKLCH migration 필요 (아래 Debt 섹션).

**Urgent / destructive 는 예외색**
- 급구 배지 · 정산 실패 · 공고 삭제 같은 critical 순간에만 빨강 계열 1회 표시.
- 세션 내 결정: 과거 `red-500` 하드코딩을 `var(--destructive)` 로 모두 치환 완료 (commit `6985e07`). destructive 는 brand 와 나란히 놓지 않는다 — **destructive 가 보이는 순간엔 주변에 brand green 을 노출하지 않음** 원칙.

**Pure black/white 금지**: 세션 기준 globals.css 는 모두 틴티드. 앞으로 `#000`/`#fff` 직접 사용 금지.

### Typography

**현재 (practical Korean-first default)**
- **Pretendard Variable** — 본문 + 헤딩 + UI label 전부. 한국 제품에서 Pretendard 는 impeccable 의 `reflex_fonts_to_reject` 반사적 선택이 아니라 **한글 본문 품질을 위한 컨텍스트적 필수 선택**. Korean 에서 Pretendard 는 Latin 에서 Inter 가 아님 — 대체 경쟁자가 실질적으로 없음. 그래서 허용.
- **Geist Mono** — **숫자 전용 (시급 · 거리 · 3분 · ₩9,500)**. `font-feature-settings: "tnum"` 으로 tabular figures. 절대 label/button 에 사용 금지 (monospace=technical reflex 방지).

**Hierarchy**
- 헤딩과 본문을 동일 family 로 쓰면 계층감이 약하므로 **weight contrast 를 강하게**: body 400, 부제 500, 헤딩 700~800 로 *최소 3-step*. 크기 차이보다 weight 차이가 한글에서 더 강하게 읽힘.
- 모듈러 스케일: 4/5-step `clamp()` fluid scale. ratio 최소 1.25.
- Cap 본문 line-length 65~75ch.

**금지**
- `Inter`, `DM Sans`, `Plus Jakarta Sans`, `Instrument Serif`, `Space Grotesk` 등 impeccable reflex list 전체 금지 — Latin 전용 surface 가 등장해도 이 리스트에서 고르지 않음.
- 한글 본문에 대문자(영문) 라벨 남발 금지. ALL CAPS 는 영문 4단어 이하 짧은 라벨에만.
- 헤딩마다 큰 아이콘 얹기 금지 (templated 느낌).
- gradient text 절대 금지 (`background-clip: text` 금지).
- monospace 로 기술 분위기 내기 금지.

### Spatial

- **4pt scale** 유지 (`4 8 12 16 24 32 48 64 96`). 토큰 이름은 semantic (`--space-sm` 식) — 현재 Tailwind v4 default spacing 이 4pt 라서 호환.
- **gap-based sibling spacing** — margin collapse 문제 없애기.
- Worker surface 는 `max-w-lg` (32rem) 모바일 폭 고정 + 좌우 `px-5 sm:px-6`. 현재 값 유지.
- Biz surface 는 `max-w-6xl` + 6 컬럼 grid 기반 정보 밀도.
- **카드 속 카드 금지.** 현재 `/home` 과 /posts 가 nested card 경향이 있음 — 표 ( `/my/settlements` 류) 를 포함해 nested surface 는 flat 으로 정리해야 함. (`/impeccable layout` 타겟).
- **asymmetry**: `/home` 같은 밀도 높은 모바일 피드에서도 전부 center 정렬 지양. 헤더 좌측 정렬 · 숫자 우측 정렬 · 카드 좌측 텍스트 + 우측 price/time 이 기본.

### Motion

- **reduced-motion 은 엄격 존중**. `@media (prefers-reduced-motion: reduce)` 에서 transform/scale/translate 제거, opacity/color fade 만 유지.
- Page-load staggered reveal **금지** (모바일 4G · 저사양 기기 컨텍스트). 한 번에 렌더.
- micro-interaction 은 "상태 변화 피드백" 에만 사용 — 버튼 press · 토글 전환 · 스와이프 refresh. 장식용 hover sparkle 금지.
- exponential easing (`ease-out-quart/quint/expo`) 기본. bounce/elastic 금지.

---

### Interaction

- **Optimistic UI 가 원칙**. 지원·찜·시간대 토글 같은 플로우는 서버 왕복을 기다리지 않고 즉시 UI 반영, 실패 시 rollback + toast. 특히 Worker 모바일은 `탭 → 0ms 반응` 을 목표.
- **Empty state 는 교육 도구** — "해당 조건의 공고가 없습니다" 같은 중립 메시지보다 "반경을 3km 로 넓혀볼까요?" 같은 행동 유도가 항상 우선.
- **Modal 지양** — 풀스크린 sheet / inline expand / dedicated page 를 먼저 고려. 현재 /biz 영역에 confirm modal 이 좀 있음, 정리 대상.

### Accessibility

- **WCAG AA 목표** (AAA 는 commercial gig marketplace 에 과잉).
- **Tap target 최소 44×44 px** 모바일 전역. 현재 `ToggleGroupItem` 기본 h-9 (36px) 는 borderline — `min-h-11` 로 보정 예정 (layout debt).
- Focus ring: `ring-brand/40` 2-3px 가시적. keyboard-only 사용자가 "어디 있는지" 를 한 눈에.
- 한국어 `word-break: keep-all` 은 전역 적용 완료 (세션 이전 commit).
- Contrast: 본문 4.5:1, 큰 헤딩 3:1. brand green (`#5bc98a`) 을 본문 배경으로 쓰면 텍스트는 `--brand-deep`(#2b8a55) 여야 4.5:1 통과 — 현재 `--brand-light + text-brand-deep` 조합이 이 룰을 지키고 있음. 유지.

### Surface tonality (Worker vs Biz 의도적 온도차)

같은 브랜드, 같은 팔레트, 같은 폰트, **다른 공간 리듬**:

| 축 | Worker (mobile, in-motion) | Biz (desktop, at-work) |
|---|---|---|
| 주 톤 | 산뜻 · 경쾌 | 단단 · 정돈 |
| 밀도 | 낮음 (한 화면 = 1 결정) | 높음 (한 화면 = 상태 파악 + 행동) |
| radius | `rounded-2xl` ~ `rounded-[28px]` (soft) | `rounded-lg` ~ `rounded-xl` (tight) |
| motion | 살짝 있는 micro-feedback | 거의 없음 — instant swap |
| 숫자 typography | Pretendard 700+ 큰 스케일 | Geist Mono tabular small |
| empty state | 친근한 action prompt | 담담한 데이터 공백 표시 |
| 브랜드 green 사용 | 강조 1회/화면 | 딱 `active nav item` 1곳 |

세션에서 심플화한 celery mark · 팔레트 라이트닝 · 기능 카피 는 두 surface 모두에 공통 적용.

## Design Principles (향후 모든 판단의 tiebreaker)

1. **가볍게 느껴지지 않으면 틀렸다.** 화면 추가·옵션 추가·설명 추가 전에 항상 "이게 Worker 의 30초 결정을 늦추나?" 를 먼저 물어본다.
2. **산뜻함은 절제로 만든다.** brand green 은 화면당 한 번. 두 번째 초록이 보이는 순간 "건강식품 브랜드" 로 미끄러진다.
3. **말장난보다 기능.** "산뜻한 샐러리 한 움큼" 을 거부했던 원칙이 앞으로도 유효. 수식어가 없어도 전달되면 그게 더 좋은 카피다.
4. **즉답하지 않는 버튼은 버그.** 눌렀는데 0ms 안에 뭔가 바뀌지 않으면 고장으로 취급 — optimistic UI + pending state 색 변화가 최소 baseline.
5. **채용 플랫폼처럼 보이는 순간 끝이다.** 알바천국·사람인 과 한 화면에 놓았을 때 "같은 카테고리 제품" 으로 보이면 디자인이 실패한 것.
